# Лабораторне заняття №14. Контрольні питання

1. CMS — це система керування контентом, яка дозволяє створювати, редагувати, публікувати й організовувати вміст сайту.
2. CMS використовують, коли потрібно швидко створювати сайт, часто оновлювати сторінки, додавати медіа й керувати контентом без ручного програмування.
3. Переваги CMS: зручна адмінпанель, теми, плагіни, розподіл прав користувачів. Застереження: безпека, оновлення, якість плагінів і залежність від обраної системи.
4. Популярні безкоштовні CMS: WordPress для блогів і універсальних сайтів, Joomla для сайтів і магазинів, Drupal для складних порталів, MODX для гнучких проєктів, OpenCart для інтернет-магазинів.
5. WordPress має просту адмінпанель, багато тем і плагінів, активну спільноту та просте встановлення.
6. Типові проблеми WordPress: несумісні плагіни, застарілі теми, ризики безпеки, повільна робота через зайві розширення.
7. Для WordPress-хостингу потрібні PHP, MySQL або MariaDB, вебсервер Apache/Nginx, HTTPS, достатньо пам'яті й можливість регулярного резервного копіювання.
8. Комерційні CMS можуть давати офіційну підтримку, готові модулі, гарантії, документацію та спеціалізовані рішення для бізнесу.
9. Вагомі критерії вибору комерційної CMS: ціна, підтримка, безпека, масштабованість, інтеграції, документація, зручність адмінпанелі.
10. Самописні CMS використовують для нестандартних проєктів, коли готові системи не підходять. Слід враховувати складність підтримки, залежність від розробника й вартість доопрацювання.
