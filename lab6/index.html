Каскад — Порядок слідування

Коли два правила мають однакову специфічність, перемагає те, що стоїть останнім у CSS-файлі.

Приклад: два h1 — однакова специфічність
h1 { color: red; }   /* 1-е правило */
h1 { color: blue; }  /* 2-е правило */
Результат:

Цей заголовок — синій (перемагає останнє)

Обидва селектори h1 мають специфічність 0001. Виграє правило, що стоїть нижче у файлі — color: blue.

Порядок підключення файлів CSS
<!-- reset.css підключено першим -->
<link rel="stylesheet" href="reset.css">
<!-- style.css підключено другим —
     його правила переважають -->
<link rel="stylesheet" href="style.css">

Файл, підключений пізніше, перекриває попередній за умови однакової специфічності.

Пріоритет джерел стилів
1
Стилі браузера User Agent Stylesheet — найнижчий пріоритет
2
Зовнішній CSS <link rel="stylesheet" href="style.css">
3
Внутрішній CSS <style> ... </style>
4
Вбудований CSS style="color: red" — найвищий пріоритет

Спадкування — inherit, initial, unset, all

Деякі властивості CSS (color, font-family) успадковуються дочірніми елементами. Інші (width, border, margin) — ні. Керувати спадкуванням дозволяють ключові слова.

inherit

Батько: червоний колір

Дитина: inherit — успадковує червоний батька

.parent { color: red; }
.child-inherit {
  color: inherit; /* = red */
}
initial

Батько: синій колір

Дитина: initial — скидає до браузерного за замовч.

.parent-blue { color: blue; }
.child-initial {
  color: initial; /* = black */
}
unset

border: 2px solid red border: unset → прибирає

Колір батька фіолетовий color: unset → успадковує фіолетовий 

/* Ненаслідувана властивість: */
.unset-border { border: unset; }
/* Наслідувана властивість: */
.unset-color  { color: unset; }
all: unset
Стилізований blockquote (orange + blue border)
blockquote з all: unset — всі стилі скинуто 
blockquote {
  background: orange;
  border: 2px solid blue;
}
.bq-unset {
  all: unset; /* скидає все */
}

Конфлікт стилів — Задача з методички

Три правила конкурують за колір тексту в <p class="description"> всередині <div class="card" id="main-product">.

HTML:
<div class="card" id="main-product">
  <p class="description">Текст</p>
</div>
CSS (три правила):
/* Специфічність: 0011 */
.card p {
  color: blue;
}

/* Специфічність: 0020 */
.card .description {
  color: green;
}

/* Специфічність: 0110 — ПЕРЕМОЖЕЦЬ */
#main-product .description {
  color: red;
}

Обчислення специфічності

.card p
Клас ×1 = 10 + Тег ×1 = 1
= 0011
color: blue
.card .description
Клас ×2 = 20
= 0020
color: green
Переможець
#main-product .description
ID ×1 = 100 + Клас ×1 = 10
= 0110
color: red

Живий приклад:

Цей текст — червоний, тому що #main-product .description (0110) перемагає .card .description (0020) та .card p (0011).

Змінити результат БЕЗ !important

Щоб зробити текст зеленим — підвищуємо специфічність зеленого правила:

/* Спосіб 1: додати ID у селектор (.card → #main-product) */
#main-product.card .description {
  color: green; /* специфічність: 0120 */
}

/* Спосіб 2: додати ще один клас */
.card .description.text-content {
  color: green; /* специфічність: 0030 */
}

/* Спосіб 3: додати батьківський ID до зеленого */
#main-product .card .description {
  color: green; /* специфічність: 0120 */
}

Цей текст — зелений завдяки підвищеній специфічності #main-product.card .description (0120) — без !important.

CSS Змінні — var()

CSS-змінні (custom properties) оголошуються через --назва: значення у :root (глобально) або в будь-якому селекторі (локально). Використовуються через var(--назва, fallback).

Глобальні змінні у :root (з методички):
:root {
  --main-color:   #2563eb;   /* основний синій */
  --accent-color: #f59e0b;   /* акцентний жовтий */
  --text-color:   #333;      /* колір тексту */
  --font-main:    'PT Sans', sans-serif;

  /* Додаткові змінні в цій роботі: */
  --bg-color:        #f8edeb;
  --card-bg:         #ffffff;
  --border-radius:   10px;
  --spacing-md:      16px;
  --transition:      0.25s ease;
}

Демонстрація var()

body — шрифт та колір
body {
  font-family: var(--font-main);
  color: var(--text-color);
}

Текст шрифтом PT Sans, колір з --text-color

h1 — колір заголовку
h1 {
  color: var(--main-color);
}

Заголовок кольору --main-color

button — фон та hover
button {
  background: var(--main-color);
}
button:hover {
  background: var(--accent-color);
}
var() з fallback-значенням
/* fallback, якщо змінна не задана */
color: var(--unknown-color, gray);

border: 2px solid
  var(--border-color, #c9184a);

Цей текст сірий (fallback)

Рамка через var з fallback
Локальна змінна в компоненті
.alert {
  --alert-clr: #e63946;
  color: var(--alert-clr);
  border-color: var(--alert-clr);
}
Локальна --alert-clr перекриває глобальну
Математика з calc() + var()
:root { --base: 16px; }

h2 { font-size: calc(var(--base) * 1.5); }
h1 { font-size: calc(var(--base) * 2.5); }
.small { font-size: calc(var(--base) * 0.75); }

h2: calc(16px × 1.5) = 24px

Область видимості змінних

:root { --color: blue; } — глобальна
.block-a — бачить глобальну синю
.block-b { --color: red; } — локальна
Всередині .block-b — червона (перекрита локальною)
Ще один нащадок .block-b — теж червона

Світла / Темна тема через CSS-змінні

Перемикання теми — лише змінюємо значення змінних у [data-theme="dark"]. Жодного дублювання стилів компонентів не потрібно. Натисніть кнопку у навбарі: «Темна».

Змінні для тем:
/* Світла тема (за замовч.) */
:root, [data-theme="light"] {
  --bg:         #f8edeb;
  --surface:    #ffffff;
  --on-surface: #2d2d2d;
  --primary:    #c9184a;
  --border:     #ffb3c1;
  --muted:      #666;
}

/* Темна тема */
[data-theme="dark"] {
  --bg:         #1a1a2e;
  --surface:    #16213e;
  --on-surface: #e0e0e0;
  --primary:    #ff4d6d;
  --border:     #4a3f5c;
  --muted:      #aaa;
}
Перемикання у JavaScript:
function toggleTheme() {
  const html = document.documentElement;
  const current = html.getAttribute('data-theme');
  html.setAttribute(
    'data-theme',
    current === 'dark' ? 'light' : 'dark'
  );
}
Компоненти використовують змінні:
body {
  background: var(--bg);
  color: var(--on-surface);
}
section {
  background: var(--surface);
  border: 1px solid var(--border);
}
/* Стилі НЕ змінюються —
   тільки значення змінних! */

Набір стилів — Картки товарів з CSS-змінними

Картки побудовані повністю на CSS-змінних. Зміна однієї змінної --main-color одночасно оновлює рамку, заголовок і кнопку.

CSS для карток (точно з методички + змінні):
.card {
  border: 2px solid var(--main-color);
  padding: 16px;
  border-radius: 10px;
}
.card h3 {
  color: var(--main-color);
}
.card button {
  background: var(--main-color);
  color: white;
}
.card button:hover {
  background: var(--accent-color);
}

Відповіді на контрольні питання

1. Що таке каскад стилів?
Каскад — це алгоритм, за яким браузер визначає, яке CSS-правило застосувати до елемента, коли кілька правил конкурують. Враховуються три фактори (за зростанням пріоритету): порядок слідування → специфічність → важливість (!important).
2. Що впливає на вибір стилю?
  1. Важливість — !important перебиває все
  2. Специфічність — чим конкретніший селектор, тим він сильніший
  3. Порядок слідування — при рівній специфічності перемагає останнє правило
  4. Джерело стилів — inline > internal > external > browser default
3. Що таке специфічність?
Специфічність — це числова вага CSS-селектора, яка визначає його пріоритет при конфлікті. Чим конкретніший (точніший) селектор, тим вища його специфічність.
4. Як обчислюється специфічність?
За чотирма розрядами Inline–ID–Class–Tag:
  • style="" → 1000
  • #id → +100 за кожен
  • .class, [attr], :hover → +10 за кожен
  • div, ::before → +1 за кожен
  • *, +, >, ~, :not() → 0
5. Який селектор сильніший: .card p чи #main p?
#main p сильніший — його специфічність 0101 (ID=100 + тег=1), тоді як .card p має 0011 (клас=10 + тег=1).
6. Чому не рекомендується використовувати !important?
  • Порушує природний каскад і робить код важко дебагованим
  • Щоб перебити !important, потрібен ще один !important з вищою специфічністю
  • Призводить до «гонки специфічностей» у великих проєктах
  • Краща альтернатива — підвищити специфічність селектора
7. Що таке CSS змінні?
CSS Custom Properties (змінні) — це іменовані значення, що визначаються розробником і перевикористовуються в CSS через функцію var(). Оголошуються через подвійне тире: --назва: значення.
8. Де оголошуються глобальні змінні?
У псевдокласі :root — він відповідає кореневому елементу <html> і має найвищу специфічність серед усіх селекторів-елементів. Змінні, оголошені в :root, доступні у всьому документі.
9. Як використовувати var()?
var(--назва-змінної) — просте використання.
var(--назва-змінної, fallback) — із запасним значенням, якщо змінна не визначена.
Приклад: color: var(--primary-color, #333);
10. Які переваги змінних CSS?
  • Централізоване управління кольорами, шрифтами, відступами
  • Легке перемикання тем (light/dark) без дублювання правил
  • Змінюються динамічно через JavaScript (element.style.setProperty())
  • Підтримка спадкування та локальних перевизначень
  • Покращують читабельність та підтримку коду
