# Аналіз підходів Bootstrap та Vanilla CSS/JS

**Виконав:** КН-22, Барулін Богдан.

Bootstrap дозволяє дуже швидко створити адаптивний інтерфейс, тому що має готову сітку, кнопки, картки, форми, navbar і модальні вікна. У Bootstrap-версії потрібно менше власного CSS, а адаптивність реалізується готовими класами `col-12`, `col-md-6`, `col-lg-4`. Такий підхід зручний для типових сайтів, навчальних проєктів, адмінпанелей і швидких прототипів.

Vanilla CSS/JS потребує більше часу, тому що сітку, стани, модальне вікно, меню і стилі потрібно писати самостійно. Зате цей підхід дає більше контролю над дизайном, поведінкою елементів і розміром коду. У Vanilla-версії адаптивність реалізовано через CSS Grid, Flexbox і media queries, а інтерактивність — через `addEventListener` та `classList.toggle`.

Bootstrap доцільно використовувати, коли важлива швидкість розробки й достатньо стандартного вигляду компонентів. Vanilla CSS/JS краще підходить для унікальних інтерфейсів, де потрібна повна гнучкість, менше зайвого коду і точний контроль над усіма деталями. На практиці часто використовують комбінований підхід: Bootstrap для базової структури, а власний CSS/JS — для кастомізації.
